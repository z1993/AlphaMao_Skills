# 市场规模测算方法论

## 核心框架: TAM/SAM/SOM

```
┌─────────────────────────────────────────────┐
│              TAM (总可触达市场)              │
│    ┌─────────────────────────────────┐      │
│    │       SAM (可服务市场)           │      │
│    │    ┌───────────────────┐        │      │
│    │    │  SOM (可获取市场)  │        │      │
│    │    └───────────────────┘        │      │
│    └─────────────────────────────────┘      │
└─────────────────────────────────────────────┘
```

---

## 方法一: Top-Down (自上而下)

**适用场景**: 有权威行业报告或宏观数据

**步骤**:
1. 获取行业总规模 (TAM)
2. 按细分维度逐层筛选 (SAM)
3. 估算可获取份额 (SOM)

**示例**:
```
TAM = 中国企业软件市场 = 5000 亿元
SAM = TAM × HR SaaS 占比 (8%) × 中大型企业占比 (40%) = 160 亿元
SOM = SAM × 目标市占率 (2%) = 3.2 亿元
```

**优点**: 快速、有权威背书
**缺点**: 可能过于乐观，细分数据难获取

---

## 方法二: Bottom-Up (自下而上)

**适用场景**: 有微观数据 (客单价、目标客群数量)

**步骤**:
1. 定义目标客户画像
2. 估算目标客户数量
3. 计算单位经济价值
4. 相乘得到市场规模

**示例**:
```
目标客户 = 员工 100+ 的中国企业 = 50 万家
目标渗透率 = 30% 需要 HR SaaS = 15 万家
年均客单价 = 10 万元/年
TAM = 15 万 × 10 万 = 150 亿元
```

**优点**: 贴近实际，假设可验证
**缺点**: 数据收集耗时，需要行业经验

---

## 方法三: 三角验证 (Triangulation)

**适用场景**: 两种方法均可执行

**步骤**:
1. 分别用 Top-Down 和 Bottom-Up 计算
2. 比较两个结果
3. 分析差异原因
4. 取合理范围或加权平均

**差异分析维度**:
- Top-Down > Bottom-Up: 可能高估了细分占比
- Bottom-Up > Top-Down: 可能低估了客单价

## 数据来源建议

### 一手数据 (更可靠)
- 公司内部销售数据
- 客户调研/访谈
- 行业专家访谈

### 二手数据 (更快捷)
| 来源类型 | 示例 | 可信度 |
|----------|------|--------|
| 专业咨询 | 艾瑞、亿欧、36Kr | ⭐⭐⭐⭐ |
| 上市公司财报 | 年报、招股书 | ⭐⭐⭐⭐⭐ |
| 政府统计 | 国家统计局、工信部 | ⭐⭐⭐⭐⭐ |
| 新闻报道 | 引用需交叉验证 | ⭐⭐ |
| 专家估计 | 需注明假设 | ⭐⭐⭐ |

---

## 常见陷阱

1. **定义模糊**: "市场"边界不清，导致重复计算或遗漏
2. **乐观偏见**: SOM 估计过高 (经验值通常 1-5%)
3. **静态视角**: 忽略市场增长率和竞争动态
4. **数据陈旧**: 使用过期数据，尤其高增长行业
5. **单一来源**: 仅依赖一个数据源，不做交叉验证
