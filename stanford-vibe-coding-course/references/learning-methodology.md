# CS146S 学习方法论

本文档解释学习系统的核心方法和实践流程。

---

## 核心方法

### 1. Zettelkasten 原子笔记

**原则**:
- **原子性**: 一个笔记只包含一个概念
- **自主性**: 每个笔记独立可读，不依赖上下文
- **关联性**: 通过 `[[双向链接]]` 连接相关概念
- **用自己的话**: 不是复制粘贴，是理解后重述

**笔记结构**:
```markdown
# [概念名称]

## 一句话定义
[用最简单的语言解释]

## 详细说明
[原理、背景、为什么重要]

## 代码示例
[如果适用]

## 关联
[[相关概念1]] - 关系说明
[[相关概念2]] - 关系说明

## 我的理解
> [用自己的话 - 这是最重要的部分]
```

---

### 2. Flashcard 间隔复习

**原理**: 在即将遗忘时复习，效果最好（Ebbinghaus 遗忘曲线）

**2357 法则**:
| 复习次数 | 间隔 |
|----------|------|
| 1 | 学习当天 |
| 2 | 1天后 |
| 3 | 3天后 |
| 4 | 7天后 |
| 5 | 14天后 |
| 6 | 30天后 |

**Flashcard 格式** (Obsidian SR 插件):
```markdown
Q: MCP 的全称是什么?
A: Model Context Protocol
<!--SR:!2026-02-03,1,230-->

Q: MCP 解决什么核心问题?
A: 让 LLM 能够标准化地与外部工具交互（自描述接口）
<!--SR:!2026-02-03,1,230-->
```

**好卡片 vs 坏卡片**:
- ✅ 问题明确，答案简洁，一张卡一个知识点
- ❌ 问题太宽泛（"解释 MCP"），答案太长

---

### 3. 主动问答

**流程**:
1. 阅读完整翻译内容
2. AI 提出 2-3 个关键问题
3. 用户用自己的话回答
4. AI 点评、修正、补充
5. 整合成笔记

---

## 单次学习流程

```
0. 选择本次学习的 Reading
   ↓
1. 阅读 AI 提供的完整中文翻译 (带重点标注)
   ↓
2. 回答 AI 提出的问题 (主动问答)
   ↓
3. AI 点评你的理解，整合成 Zettelkasten 笔记
   ↓
4. 更新进度
```

## 周复习流程 (Week Finish)

当本周内容全部完成后：

1. **回顾**: 浏览本周所有笔记
2. **生成闪卡**: AI 综合本周知识点，批量生成 ~10 张核心卡片
   *格式优化*: 使用 Emoji (❓/💡) 和粗体增强可读性
3. **周总结**: 填写 Week X 总结模板

- 插件地址: https://github.com/st3v3nmw/obsidian-spaced-repetition
