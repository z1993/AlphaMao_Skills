# CS146S 作业指南（简化版）

本文档提供9个课程作业的简化版本，适合自学者独立完成。

> 💡 你可以使用任何 AI 编程工具来完成这些作业，不限于 Cursor、Claude Code 或 Warp。

---

## Week 1: Prompt Engineering 练习

**目标**: 掌握6种核心 Prompt 技术

### 任务
用任意 LLM (如 ChatGPT, Claude, 本地 Ollama) 实践以下技术：

1. **K-shot Prompting**: 给 LLM 2-3 个示例，让它学会新任务
2. **Chain-of-Thought**: 让 LLM "一步步思考"解决复杂问题
3. **Tool Calling**: 让 LLM 调用外部函数/API
4. **Self-Consistency**: 多次采样取最一致的答案
5. **RAG**: 检索文档后生成回答
6. **Reflexion**: 让 LLM 反思并改进自己的答案

### 交付物
为每种技术写一个实际的 prompt 示例和输出结果

### 自评问题
- 哪种技术最有效？为什么？
- 遇到了什么问题？如何解决的？

## Week 2: AI IDE 初体验

**目标**: 熟悉 AI 编程助手

### 任务
用任意 AI 编程工具扩展一个简单应用（可以是 Todo、笔记等）：

1. 添加一个 LLM 驱动的功能（如智能分类、摘要）
2. 为新功能写单元测试
3. 用 AI 生成 README 文档

### 交付物
- 代码仓库链接
- 截图展示 AI 帮助你完成任务的过程

### 自评问题
- AI 在哪些地方帮助最大？
- 哪些地方需要人工干预？

## Week 3: 构建 MCP Server ⭐

**目标**: 理解 MCP 协议，动手实现一个 Server

### 任务
为你常用的 API（天气、GitHub、Notion 等）构建 MCP Server：

1. 实现至少2个工具 (tools)
2. 处理错误和超时
3. 写清晰的使用文档

### 交付物
- 代码仓库
- README 包含：安装、运行、使用示例

### 自评问题
- 为什么 MCP 比直接调 API 更好？
- 遇到了什么挑战？

## Week 4: Claude Code 自动化 ⭐

**目标**: 构建开发工作流自动化

### 任务
使用任意 AI Agent 工具，创建2个以上自动化：

**选项**：
- 自定义命令/快捷方式
- 项目配置文件（如 CLAUDE.md, .cursor/rules）
- 多 Agent 协作工作流
- MCP Server 集成

### 交付物
- 自动化配置文件
- 使用演示（截图或简短说明）

### 自评问题
- 自动化节省了多少时间？
- 自动化的边界在哪里？

## Week 6: 安全扫描与修复 ⭐

**目标**: 建立 AI 代码安全意识

### 任务
对任意项目进行安全扫描并修复漏洞：

1. 使用 Semgrep、Snyk 或其他 SAST 工具扫描
2. 选择3个问题，用 AI 帮助修复
3. 验证修复有效

### 交付物
- 扫描报告截图
- 修复前后对比

### 自评问题
- SAST 发现了哪类问题？
- AI 修复建议靠谱吗？

## Week 7: AI 代码审查

**目标**: 理解 AI 在 Code Review 中的作用

### 任务
1. 完成一个功能开发并提交 PR
2. 自己先人工审查
3. 使用 AI 工具审查同一 PR
4. 对比人工 vs AI 审查结果

### 交付物
- PR 链接
- 人工审查 vs AI 审查对比表

### 自评问题
- AI 审查漏了什么？多发现了什么？
- 你会信任 AI 审查结果吗？

## Week 8: 多技术栈应用构建

**目标**: 体验 AI 生成应用的能力

### 任务
用3种不同方式构建同一个简单应用（CRUD 功能）：

1. 使用 AI 应用生成器（如 bolt.new, v0.dev）
2. 使用另一个 AI 生成器或框架
3. 传统手写（可用 AI 辅助）

### 交付物
- 3个版本的代码
- 各版本优缺点对比

### 自评问题
- AI 生成的代码质量如何？
- 什么情况适合用 AI 生成器？

## 作业评价系统

完成作业后，可以与 AI 教练进行评价互动：

**触发命令**: `评价作业 Week X`

AI 会根据以下维度评价：
- **完成度** (0-30分): 是否完成核心要求
- **理解深度** (0-30分): 是否理解原理而非只是照做
- **创造性** (0-20分): 是否有自己的思考和拓展
- **文档质量** (0-20分): 说明是否清晰

**评分标准**:
- 90-100: 优秀，可以作为示例
- 75-89: 良好，掌握了核心内容
- 60-74: 合格，需要加强理解
- <60: 需要重做
