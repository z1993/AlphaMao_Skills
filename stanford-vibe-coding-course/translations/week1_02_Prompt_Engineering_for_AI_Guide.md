# AI 提示工程指南

**大型语言模型（LLMs）** 的兴起为通过自然语言与计算机交互带来了令人兴奋的可能性。然而，要充分利用这些强大 AI 模型的潜力，需要一项关键技能：**提示工程（Prompt Engineering）**。

这个新兴领域专注于设计有效的提示，以解锁 LLM 的能力，使其能够理解意图、遵循指令并生成预期的输出。随着我们在各种应用中越来越多地与 AI 互动，提示工程在确保准确、相关和安全的交互中发挥着至关重要的作用。

---

## 什么是提示工程？

**提示工程（Prompt Engineering）** 是一门设计和优化提示的艺术与科学，旨在引导 AI 模型（特别是 LLM）生成预期的响应。通过精心制作提示，你可以为模型提供上下文、指令和示例，帮助它理解你的意图并以有意义的方式做出回应。

可以把它想象成给 AI 提供一张路线图，引导它通向你心目中的特定输出结果。

---

## 什么是 AI 提示？

在 AI 的语境下，**提示（Prompt）** 是你提供给模型的输入，用于引出特定的响应。

这种输入可以采取多种形式，从简单的问题或关键词，到复杂的指令、代码片段，甚至创意写作样本。提示的有效性直接影响 AI 输出的质量和相关性。

---

## 提示工程需要什么？

有几个关键要素有助于实现有效的提示工程。掌握这些要素可以让你有效地与 AI 模型沟通并释放其全部潜力。

### 1. 提示格式 (Prompt format)
提示的结构和风格在引导 AI 响应方面起着重要作用。不同的模型可能对特定的格式反应更好，例如：自然语言问题、直接命令或具有特定字段的结构化输入。了解模型的能力和偏好格式对于制作有效的提示至关重要。

### 2. 上下文和示例 (Context and examples)
在提示中提供上下文和相关示例有助于 AI 理解所需的任务并生成更准确、相关的输出。例如，如果你在寻求一个创意故事，包含几句描述所需语气或主题的句子可以显著改善结果。

### 3. 微调和适应 (Fine-tuning and adapting)
使用定制的提示在特定任务或领域对 AI 模型进行微调可以增强其性能。此外，根据用户反馈或模型输出调整提示，可以随着时间的推移进一步改善模型的响应。

### 4. 多轮对话 (Multi-turn conversations)
为多轮对话设计提示允许用户与 AI 模型进行连续且具有上下文感知的交互，从而增强整体用户体验。

---

## 提示的类型

AI 中使用了多种类型的提示，每种都有特定的用途：

### 1. 直接提示 (Direct prompts / Zero-shot)
零样本（Zero-shot）提示涉及向模型提供直接的指令或问题，而不提供任何额外的上下文或示例。
* **示例**：创意生成（让模型生成创意或头脑风暴解决方案）、摘要或翻译。

### 2. 单样本、少样本和多样本提示 (One-, few- and multi-shot prompts)
这种方法涉及在呈现实际提示之前，向模型提供一个或多个所需**输入-输出对**的示例。这可以帮助模型更好地理解任务并生成更准确的响应。

### 3. 思维链提示 (Chain of Thought Prompts, CoT)
思维链（CoT）提示鼓励模型将复杂的推理分解为一系列中间步骤，从而产生更全面、结构更良好的最终输出。

### 4. 零样本思维链提示 (Zero-shot CoT Prompts)
结合了思维链提示和零样本提示，要求模型执行推理步骤，通常能产生更好的输出。

## 提示工程的用例和示例

以下是一些具体的示例和用例，展示了提示工程如何帮助产生定制化和相关的输出。

### 1. 语言和文本生成

| 场景 | 指令 | 示例提示 |
| :--- | :--- | :--- |
| **创意写作** | 制作指定体裁、语气、风格和情节点的提示，以引导 AI 生成引人入胜的叙述。 | “写一个短篇故事，讲述一个年轻女子在她阁楼里发现了一个魔法传送门。” |
| **摘要** | 为 AI 提供文本，并指示其生成捕捉关键信息的简洁摘要。 | “总结以下关于气候变化的新闻文章的要点。” |
| **翻译** | 指定源语言和目标语言，使 AI 能够在保留含义和上下文的同时准确翻译文本。 | “将以下文本从英语翻译成西班牙语：'The quick brown fox jumps over the lazy dog.'” |
| **对话** | 设计模拟对话的提示，允许 AI 生成模仿人类互动并保持上下文的响应。 | “你是一个友好的聊天机器人，帮助用户解决电脑问题。回应用户的查询：'我的电脑开不了机。'” |

### 2. 问答 (Question Answering)

| 场景 | 指令 | 示例提示 |
| :--- | :--- | :--- |
| **开放式问题** | 制定鼓励 AI 基于其知识库提供全面且信息丰富答案的提示。 | “解释量子计算的概念及其对未来技术的潜在影响。” |
| **具体问题** | 设计针对特定信息的提示，使 AI 能够从提供的上下文或其内部知识库中检索精确答案。 | “法国的首都是哪里？” 或 “根据提供的文本，森林砍伐的主要原因是什么？” |
| **多项选择题** | 呈现带有选项的提示，促使 AI 分析并基于其对上下文的理解选择最合适的答案。 | “谁写了《哈利·波特》系列？A) J.R.R. 托尔金, B) J.K. 罗琳, C) 斯蒂芬·金” |
| **假设性问题** | 制作探索假设情境的提示，允许 AI 推理、推测并提供潜在的结果或解决方案。 | “如果人类能够以光速旅行，会发生什么？” |
| **基于观点的问题** | 设计引出 AI 对特定话题的视角或观点的提示，鼓励它为其立场提供推理和理由。 | “你认为人工智能最终会超越人类智能吗？为什么或为什么不？” |

### 3. 代码生成 (Code Generation)

| 场景 | 指令 | 示例提示 |
| :--- | :--- | :--- |
| **代码补全** | 为 AI 提供部分代码片段，并提示它根据上下文和编程语言建议或完成剩余代码。 | “写一个 Python 函数来计算给定数字的阶乘。” |
| **代码翻译** | 指定源编程语言和目标编程语言，使 AI 能够在保留功能和语法的同时翻译代码。 | “将以下 Python 代码翻译成 JavaScript: `def greet(name): print('Hello,', name)`” |
| **代码优化** | 提示 AI 分析现有代码并建议在效率、可读性或性能方面的改进。 | “优化以下 Python 代码以减少其执行时间。” |
| **代码调试** | 为 AI 提供包含错误的代码，并提示它识别并建议针对已识别问题的潜在解决方案。 | “调试以下 Java 代码并解释为什么它会抛出 NullPointerException。” |

### 4. 图像生成 (Image Generation)

| 场景 | 指令 | 示例提示 |
| :--- | :--- | :--- |
| **写实图像** | 制作详细描述所需图像（包括物体、风景、光照和风格）的提示，以生成逼真且高质量的图像。 | “一张日落时分海洋的写实照片，棕榈树在天空的映衬下形成剪影。” |
| **艺术图像** | 设计指定艺术风格、技巧和题材的提示，引导 AI 创作模仿特定艺术运动或唤起特定情感的图像。 | “一幅印象派画作，描绘繁忙的城市街道，人们在雨中撑着伞行走。” |
| **抽象图像** | 制定鼓励 AI 利用形状、颜色和纹理生成开放解释的图像的提示，以唤起感觉或概念。 | “一张代表希望概念的抽象图像，使用明亮的颜色和流动的形状。” |
| **图像编辑** | 为 AI 提供现有图像并指定所需的修改，使其能够根据给定的指令编辑和增强图像。 | “将这张照片的背景更改为星空，并添加一轮满月。” 或 “从这张图片中移除那个人，并用一只猫代替。” |

## 编写更好提示的策略

开发有效的提示需要一种战略性的方法。考虑以下策略来增强你的提示：

### 1. 提供上下文和背景信息

| 策略 | 提示示例 |
| :--- | :--- |
| **包含相关事实和数据** | “鉴于全球气温自工业化前时代以来已上升 1 摄氏度，讨论其对海平面上升的潜在后果。” |
| **引用特定来源或文档** | “根据所附的财务报告，分析该公司过去五年的盈利能力。” |
| **定义关键术语和概念** | “用适合非技术受众的简单术语解释量子计算的概念。” |

### 2. 使用少样本提示 (Few-Shot Prompting)

| 策略 | 提示示例 |
| :--- | :--- |
| **提供几个所需输入-输出对的示例** | 输入：“猫” 输出：“一种长着胡须的小型毛茸茸哺乳动物。”<br>输入：“狗” 输出：“一种以忠诚著称的驯化犬科动物。”<br>提示：“大象” |
| **演示所需的风格或语气** | 示例 1（幽默）：“这位政治家的演讲太无聊了，简直能治好失眠。”<br>示例 2（正式）：“这位政要发表了一次既丰富又有吸引力的演说。”<br>提示：“写一个句子描述这位喜剧演员的单口相声表演。” |
| **展示所需的详细程度** | 示例 1（简短）：“这部电影是关于一个小男孩和一个外星人成为朋友的故事。”<br>示例 2（详细）：“这部科幻电影讲述了艾略特的故事，一个孤独的男孩发现了一个被困在地球上的外星人，并与其建立了独特的联系。”<br>提示：“总结你刚刚读完的小说的情节。” |

## 关键术语

| 英文 | 中文 | 说明 |
| :--- | :--- | :--- |
| **Prompt Engineering** | 提示工程 | 设计和优化输入以引导 AI 模型生成特定输出的学科。 |
| **Large Language Models (LLMs)** | 大型语言模型 | 在海量文本数据上训练的 AI 模型，能够理解和生成人类语言。 |
| **Zero-shot Prompting** | 零样本提示 | 不提供任何示例，直接给出指令让模型执行任务。 |
| **Few-shot Prompting** | 少样本提示 | 在提示中包含少量示例（输入-输出对）以指导模型。 |
| **Chain of Thought (CoT)** | 思维链 | 一种提示技术，鼓励模型展示推理步骤以解决复杂问题。 |
| **Multi-turn Conversations** | 多轮对话 | 用户与 AI 之间连续的、有上下文关联的互动过程。 |
| **Context** | 上下文 | 提供给模型的背景信息，帮助其更准确地理解任务。 |
