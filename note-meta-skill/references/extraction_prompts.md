# 知识萃取 Prompts

本文件包含用于从 NotebookLM 中萃取核心知识的结构化提问模板。

---

## 1. Workflow 萃取

```
请根据知识库中的内容，提取完整的工作流程：

1. 这套方法/技能的完整工作流是什么？
2. 每一步具体要做什么？（步骤名称 + 具体动作）
3. 每一步的输入和输出是什么？
4. 步骤之间有什么依赖关系？

请用以下格式回答：

## 工作流

### 步骤 1: [步骤名称]
- **具体动作**: ...
- **输入**: ...
- **输出**: ...

### 步骤 2: [步骤名称]
...
```

---

## 2. Principles 萃取

```
请根据知识库中的内容，提取核心原则和方法论：

1. 这套方法背后的核心原则是什么？（不只是"做什么"，更要回答"为什么这样做"）
2. 有哪些关键的指导思想或理念？
3. 有没有常见的误区需要避免？
4. 这些原则如何相互支撑、形成体系？

请用以下格式回答：

## 核心原则

### 原则 1: [原则名称]
- **内容**: ...
- **Why（为什么）**: ...
- **How（如何应用）**: ...

### 原则 2: [原则名称]
...

## 常见误区
- 误区1: ...
- 误区2: ...
```

---

## 3. Templates 萃取

```
请根据知识库中的内容，提取可复用的模板、框架或清单：

1. 有没有现成的模板可以直接使用？（如：文章模板、清单、框架图）
2. 有没有标准化的结构可以参考？
3. 有没有常用的检查清单或评估标准？

请用以下格式回答：

## 可复用模板

### 模板 1: [模板名称]
**用途**: ...
**内容**:
```
[模板内容]
```

### 模板 2: [模板名称]
...

## 检查清单
- [ ] 检查项1
- [ ] 检查项2
```

---

## 4. Scripts 萃取

```
请根据知识库中的内容，探测是否涉及代码、脚本或自动化工具：

1. 这套方法是否涉及任何代码或脚本？
2. 是否有推荐的自动化工具或软件？
3. 如果有代码，是什么语言？核心逻辑是什么？

请用以下格式回答：

## 脚本/工具

### 是否涉及脚本: [是/否]

### 相关工具
- 工具1: [名称] - [用途]
- 工具2: ...

### 代码片段（如有）
```[语言]
[代码内容]
```

如果完全不涉及代码或自动化，请明确回答"无脚本相关内容"。
```

---

## 5. 综合萃取（可选）

如果时间紧迫，可以用一个综合 Prompt 一次性萃取：

```
请根据知识库中的所有内容，帮我萃取以下四个维度的核心知识：

1. **Workflow（工作流）**: 完整的步骤流程是什么？每一步具体做什么？
2. **Principles（原则）**: 核心的指导思想和原则是什么？为什么要这样做？
3. **Templates（模板）**: 有没有可以直接复用的模板或框架？
4. **Scripts（脚本）**: 是否涉及代码或自动化工具？

请按维度分章节回答，尽量详细、结构化。
```

---

## 使用建议

1. **优先使用分步萃取**：依次执行 1→2→3→4，质量更高
2. **综合萃取作为备选**：仅在时间紧迫时使用
3. **追问深挖**：如果某个维度回答不够详细，可以追问
4. **保存萃取结果**：将每次 `notebooklm ask` 的结果保存，用于后续 Skill 封装
