# Dan Koe 写作方法论

基于 Dan Koe 的写作体系，指导你从选题到发布的完整写作流程。

## 适用场景

- 写社交媒体帖子（Twitter/LinkedIn）
- 写 Newsletter / 博客文章
- 写视频脚本
- 需要结构化写作指导时

---

## 写作 SOP（标准作业程序）

### 1. 选题：观点与视角

**核心策略**：
- **一人利基**: 结合 2-3 个核心兴趣，解决你**过去的问题**。最好的读者是"旧版自己"。
- **愿景 vs 反愿景**:
  - 反愿景（敌人）：你讨厌/避免什么？
  - 愿景（目的地）：你要带读者去哪里？
  - **选题公式**: 如何从 [反愿景] → [愿景]

**具体动作**：
- **创意博物馆**: 随时捕捉灵感（书、推文、生活），不依赖记忆。
- **折射原理**: 通过你的独特视角"折射"通用想法，形成新观点。

---

### 2. 开头：钩子 (Hook)

钩子必须包含：**关联性**、**痛点** 或 **好奇心**。

**钩子公式**（选一个）：
1. **大问题 + 负面后果**: "如果你继续忽视 [问题]，你将在 [时间] 后陷入 [糟糕结局]"
2. **大想法 + 预期打破**: "大多数人认为 [流行观点] 正确，但这正是他们失败的原因"
3. **If/Then**: "如果你想 [渴望的结果]，那么你必须停止 [常见错误]"
4. **残酷真相**: 用短句陈述反直觉事实

---

### 3. 正文：APAG 框架

| 阶段 | 名称 | 内容 |
|------|------|------|
| **A** | Attention | 钩子/大想法，用痛点开场 |
| **P** | Perspective | 树立敌人：描述大众的错误观点，放大痛点，用个人故事增强共鸣 |
| **A** | Advantage | 提出新视角/解决方案，教育论证，展示采纳后的美好未来 |
| **G** | Gamify | 提供步骤/挑战，让方案像游戏任务一样清晰 |

**简化版 (BPAS)**：Big Idea → Problem → Amplify → Solution

---

### 4. 结尾：游戏化与行动

1. **列出步骤**: Step 1, Step 2, Step 3
2. **创建挑战**: "今天试着做 X，看看会发生什么"
3. **CTA**: 引导读者进入生态系统（订阅/购买）

## 写作心法

| 心法 | 说明 |
|------|------|
| **模仿架构** | 拆解喜欢的创作者文章结构，用你的话题填入（模仿架构，非内容） |
| **给朋友发短信** | 想象写给迷茫的朋友/年轻的自己，消除专家腔 |
| **6步思考法** | ①选想法 ②问3个问题 ③写初稿 ④像路人一样读 ⑤像杠精一样质疑 ⑥格式化 |
| **整理精神熵** | 写不清楚=没想清楚，先写下来再理顺 |
| **为分享而写** | 每段话问：独立发推会被转发吗？追求高密度金句 |

## 30 分钟写作清单

当用户请求帮助写作时，按以下流程执行：

1. **选题**: 从笔记挑一个能解决"旧版自己"痛苦的概念
2. **Hook**: 用"反直觉观点"或"放大痛点"写第一句
3. **Body**:
   - 旧观念（敌人）是什么？为什么错？
   - 新观念（英雄）是什么？为什么对？
4. **End**: 变成 3 步行动清单
5. **Review**: 像路人重读，删无聊词，确保像发短信一样自然

## 示例

**用户请求**: "帮我写一篇关于'为什么大多数人学不会编程'的文章"

**执行流程**:
1. **选题确认**: 反愿景=半途而废、自我怀疑；愿景=独立解决问题、获得自由
2. **Hook**: "大多数人学编程失败，不是因为不够聪明，而是因为他们用了错误的方法。"
3. **Body (APAG)**:
   - P: 描述"看教程→抄代码→遇到问题→放弃"的恶性循环
   - A: 提出"项目驱动学习"的新视角
4. **End**: 列出 3 步入门计划 + CTA（订阅获取项目清单）
5. **Review**: 删除废话，确保金句密度
