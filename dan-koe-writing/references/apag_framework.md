# APAG 框架详解

## 框架概述

APAG 是 Dan Koe 的核心写作结构，引导读者从"被吸引"到"行动"。

```
Attention → Perspective → Advantage → Gamify
```

---

## A - Attention (注意力)

**目的**: 在 3 秒内抓住读者

**操作**:
- 使用钩子公式（见 [hook_formulas.md](hook_formulas.md)）
- 必须包含：关联性 + 痛点 或 好奇心

**检查**: 读者会不会想"这说的就是我"？

---

## P - Perspective (视角)

**目的**: 树立敌人，放大痛点

**操作**:
1. 描述大众普遍持有的**错误观点**（敌人）
2. 详细描述这个错误带来的**痛苦**
3. 用**个人故事**增强共鸣

**模板**:
> "我曾经也以为 [错误观点]……直到我发现 [转折]……"

**检查**: 读者会不会觉得"这个人理解我"？

---

## A - Advantage (优势)

**目的**: 提出新视角，展示解决方案

**操作**:
1. 提出你的**新观点/解决方案**（英雄）
2. 解释**为什么**你的方法更好
3. 提供逻辑、案例或概念支持
4. 展示采纳新视角后的**美好未来**

**检查**: 读者会不会觉得"原来可以这样"？

---

## G - Gamify (游戏化)

**目的**: 将方案变成可执行的行动

**操作**:
1. 将解决方案拆解为 **Step 1, 2, 3**
2. 创建一个**小挑战**（今天试试 X）
3. 添加 **CTA**（订阅/购买/关注）

**检查**: 读者会不会觉得"我知道下一步该做什么"？

---

## 简化版: BPAS

适用于较短内容（推文、短帖）

| 阶段 | 内容 |
|------|------|
| **B** | Big Idea - 核心概念 |
| **P** | Problem - 痛点 |
| **A** | Amplify - 放大后果 |
| **S** | Solution - 解决方案 |
